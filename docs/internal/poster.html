维度说明

🤖 控制器配置

  • NoControl — 零动作基线，暴露无控动态
  • PD — 经典反馈 (Kp=5.0, Kd=2.0)，动作裁剪到 [-1,1]
  • PPO — MlpPolicy, lr=3e-4, gamma=0.99, 2×64隐藏层
  • PPO+安全 — PPO + 一步前向预测安全过滤器
  • PPO-随机化 — 域随机化训练，每回合随机化物理参数

🔬 评估协议

  • 种子数：3（seed 0, 1, 2）
  • 训练步数：500,000 / 种子 / 控制器
  • 场景数：4（正常/强/可变/OOD）
  • 评估回合：20 回合 / 场景
  • 指标：avg_return, mean_abs_theta, failure_rate, control_energy
  • 工具栈：Gymnasium, NumPy, Stable-Baselines3, PyTorch

💡 关键发现

1
经典PD在低维线性系统中具有解析接近最优的性质。这不是RL的失败——这科学上重要的"没有免费午餐"定理的实证验证。对于低维类线性的控制系统，PD在数学上已接近最优解。
2
PPO存在极端的种子依赖性问题。种子0在所有变体的可变风和OOD场景中灾难性失败（100%故障率），而种子1-2有时能与PD竞争。这意味着单种子RL评估结论完全不可靠，揭示了RL部署的根本性风险。
3
安全过滤PPO在OOD场景中显示潜力。以18%回报优势险胜PD，但伴随更高故障率（40% vs 35%）和近乎恒定的安全干预（93.5%干预率），实际价值仍需审慎评估。
4
RL的价值体现在高维、非线性或多目标场景中。本项目的发现恰恰证明了诚实评估的重要性——当经典方法优于学习方法时，如实报告同样是高质量的科研贡献。
